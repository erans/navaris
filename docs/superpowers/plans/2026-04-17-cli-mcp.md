# CLI improvements & MCP server — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add an MCP server (stdio binary + embedded HTTP endpoint in navarisd) that exposes navaris's sandbox/session/snapshot/image surface as MCP tools, plus targeted improvements to the existing CLI.

**Architecture:** A new `internal/mcp` package holds tool registrations (schemas, descriptions, handlers) shared by both transports. All tool handlers call back through `pkg/client.Client` over HTTP — no direct service calls — so the MCP can never bypass auth, validation, or audit logging that lives on the REST handlers. A new `cmd/navaris-mcp` binary wraps the package for stdio. The embedded transport mounts an `mcp.NewStreamableHTTPHandler` at `/v1/mcp` in `navarisd`.

**Tech Stack:** Go 1.26, Cobra (CLI), `github.com/modelcontextprotocol/go-sdk` v1.5.0+ (MCP), `nhooyr.io/websocket` (CLI attach), existing `pkg/client` HTTP client.

**Spec:** [docs/superpowers/specs/2026-04-17-cli-mcp-design.md](../specs/2026-04-17-cli-mcp-design.md)

## Spec adjustments discovered during exploration

These are **deviations from the spec** the engineer should be aware of before starting:

1. **`image_list` tool** does not accept `project_id`. The server-side `GET /v1/images` accepts only `name` and `architecture` filters. The MCP `image_list` tool takes optional `name?` and `architecture?` instead.
2. **`snapshot_list` tool** is sandbox-scoped: requires `sandbox_id`. There is no project-wide snapshot listing on the server. The optional `project_id?` from the spec is dropped.
3. **`sandbox_exec` `stdin` field deferred.** Adding stdin requires changes across `domain.Provider.Exec`, the mock/Incus/Firecracker provider implementations, and the Firecracker in-guest agent. Out of scope here. The MCP `sandbox_exec` tool exposes `command`, `env`, `work_dir`, and `timeout_seconds` (server already supports `env` and `work_dir`).
4. **`session_get` MCP tool exists but has no per-session GET in the existing client.** `pkg/client.Client.GetSession` does exist (`/v1/sessions/{id}`); just confirming it's used.
5. **`DestroySession` returns `error`, not an operation.** Session destroy is synchronous; no `wait` parameter needed on `session_destroy`.
6. **Snapshot delete and image delete on the server return Operation envelopes.** `snapshot_delete` and (deferred — not in scope) image deletion are async — they need wait support.
7. **JSON keys** in existing `pkg/client` types use PascalCase for response bodies (`ProjectID`, `SandboxID`, …) and snake_case for request bodies. The MCP tools return the existing client types as-is — this means agents see `{"ProjectID": "...", "Name": "..."}`. If we want snake_case at the MCP boundary, that's a follow-up.

// parseEnvFlags converts ["KEY=value", ...] into a map. The first '=' is the
// separator so values can contain additional '=' characters. Returns an error
// for entries without '=' or with empty keys.
func parseEnvFlags(items []string) (map[string]string, error) {
	if len(items) == 0 {
		return nil, nil
	}
	out := make(map[string]string, len(items))
	for _, kv := range items {
		i := strings.IndexByte(kv, '=')
		if i < 0 {
			return nil, fmt.Errorf("--env %q: missing '=' separator", kv)
		}
		key := kv[:i]
		if key == "" {
			return nil, fmt.Errorf("--env %q: empty key", kv)
		}
		out[key] = kv[i+1:]
	}
	return out, nil
}
```

func pollSandboxState(ctx context.Context, c *client.Client, sandboxID, state string, interval time.Duration) (*client.Sandbox, error) {

		}
	}

package client

import (
	"context"

	"net/http"
	"net/url"
	"strings"

	"nhooyr.io/websocket"

// AttachConn is a thin wrapper around a WebSocket connected to a sandbox's
// attach endpoint. Use Read to receive output bytes from the sandbox and
// Write to send input bytes. Close terminates the attach.
type AttachConn struct {
	ws  *websocket.Conn
	ctx context.Context
}

// Read returns the next batch of output bytes from the sandbox. Returns
// io.EOF when the WebSocket closes cleanly.
func (a *AttachConn) Read(p []byte) (int, error) {
	_, data, err := a.ws.Read(a.ctx)
	if err != nil {
		var ce websocket.CloseError
		if asErr := errorsAs(err, &ce); asErr {
			return 0, io.EOF
		}
		return 0, err
	}
	n := copy(p, data)
	if n < len(data) {
		// Caller's buffer is too small for the full frame. Returning what
		// fit is fine — the consumer should call Read again for the rest;
		// however, we lose the tail. Use a buffered reader pattern instead
		// in callers that need exact stream semantics.
		return n, nil
	}
	return n, nil
}

// Write sends input bytes to the sandbox.
func (a *AttachConn) Write(p []byte) (int, error) {
	if err := a.ws.Write(a.ctx, websocket.MessageBinary, p); err != nil {
		return 0, err
	}
	return len(p), nil
}

// Resize sends a JSON resize control frame.
func (a *AttachConn) Resize(cols, rows int) error {
	msg, _ := json.Marshal(map[string]any{
		"type": "resize",
		"cols": cols,
		"rows": rows,
	})
	return a.ws.Write(a.ctx, websocket.MessageText, msg)
}

// Close terminates the attach.
func (a *AttachConn) Close() error {
	return a.ws.Close(websocket.StatusNormalClosure, "client closed")
}

// AttachSandbox opens a WebSocket to /v1/sandboxes/{id}/attach. If sessionID
// is non-empty it is passed as the ?session query parameter.
func (c *Client) AttachSandbox(ctx context.Context, sandboxID, sessionID string) (*AttachConn, error) {
	u, err := url.Parse(c.baseURL)
	if err != nil {
		return nil, fmt.Errorf("parse base url: %w", err)
	}
	switch u.Scheme {
	case "http":
		u.Scheme = "ws"
	case "https":
		u.Scheme = "wss"
	}
	u.Path = strings.TrimRight(u.Path, "/") + fmt.Sprintf("/v1/sandboxes/%s/attach", sandboxID)
	q := u.Query()
	if sessionID != "" {
		q.Set("session", sessionID)
	}
	if c.token != "" {
		q.Set("token", c.token)
	}
	u.RawQuery = q.Encode()

	hdr := http.Header{}
	if c.token != "" {
		hdr.Set("Authorization", "Bearer "+c.token)
	}
	conn, _, err := websocket.Dial(ctx, u.String(), &websocket.DialOptions{HTTPHeader: hdr})
	if err != nil {
		return nil, fmt.Errorf("dial attach: %w", err)
	}
	return &AttachConn{ws: conn, ctx: ctx}, nil
}

// errorsAs is a tiny indirection so the file does not need an extra import
// alias for errors.As. Inlined here to avoid touching the whole package.
func errorsAs(err error, target any) bool {
	for {
		if x, ok := target.(*websocket.CloseError); ok {
			if ce, ok := err.(websocket.CloseError); ok {
				*x = ce
				return true
			}
		}
		// Not strictly required for our use; for simplicity, no unwrap chain.
		return false
	}
}

// runAttachLoop wires os.Stdin/os.Stdout to the attach connection. SIGWINCH
// triggers a Resize frame. The function returns when stdin closes, the WS
// closes, or the context is cancelled.
func runAttachLoop(ctx context.Context, conn *client.AttachConn) error {
	// Set raw mode on stdin so keystrokes pass through.
	stdin := int(os.Stdin.Fd())
	oldState, err := termSetRaw(stdin)
	if err != nil {
		return fmt.Errorf("set raw mode: %w", err)
	}
	defer termRestore(stdin, oldState)

	// Send the initial resize.
	if cols, rows, err := termSize(stdin); err == nil {
		_ = conn.Resize(cols, rows)
	}

	// SIGWINCH handler.
	winch := make(chan os.Signal, 1)
	signalNotify(winch, sigwinch())
	defer signalStop(winch)
	go func() {
		for range winch {
			if cols, rows, err := termSize(stdin); err == nil {
				_ = conn.Resize(cols, rows)
			}
		}
	}()

	// Stdin → conn.
	go func() {
		buf := make([]byte, 4096)
		for {
			n, err := os.Stdin.Read(buf)
			if n > 0 {
				if _, werr := conn.Write(buf[:n]); werr != nil {
					return
				}
			}
			if err != nil {
				return
			}
		}
	}()

	// Conn → stdout.
	buf := make([]byte, 16384)
	for {
		n, err := conn.Read(buf)
		if n > 0 {
			_, _ = os.Stdout.Write(buf[:n])
		}
		if err == io.EOF {
			return nil
		}
		if err != nil {
			return err
		}
	}
}

```go
package mcp

import (
	"context"
	"fmt"
	"time"

	"github.com/navaris/navaris/pkg/client"
)

// progressResponse is the shape returned when a per-tool timeout elapses but
// the underlying operation is still running. Returned as a non-error result so
// the agent can decide whether to keep polling via operation_get.
type progressResponse struct {
	OperationID string `json:"operation_id"`
	Status      string `json:"status"`
	Note        string `json:"note"`
}

// waitForOpAndFetch waits for op to reach a terminal state (or until timeout
// elapses), then calls fetch to return the up-to-date resource.
//
//   - If op is already succeeded, fetch is called immediately.
//   - If op fails, returns an error including the operation_id and error_text.
//   - If op is cancelled, returns an error.
//   - If timeout elapses while op is still running, returns a non-error
//     progress payload so the agent can poll operation_get.
func waitForOpAndFetch(
	ctx context.Context,
	c *client.Client,
	op *client.Operation,
	timeout time.Duration,
	fetch func() (any, error),
) (any, error) {
	if op.State == client.OpSucceeded {
		return fetch()
	}
	if op.State == client.OpFailed {
		return nil, fmt.Errorf("operation %s failed: %s", op.OperationID, op.ErrorText)
	}
	if op.State == client.OpCancelled {
		return nil, fmt.Errorf("operation %s was cancelled", op.OperationID)
	}

	final, err := c.WaitForOperation(ctx, op.OperationID, &client.WaitOptions{Timeout: timeout})
	if err != nil {
		// Treat timeout as in-progress, not an error.
		return progressResponse{
			OperationID: op.OperationID,
			Status:      "running",
			Note:        "still in progress, poll operation_get",
		}, nil
	}
	if final.State == client.OpFailed {
		return nil, fmt.Errorf("operation %s failed: %s", final.OperationID, final.ErrorText)
	}
	if final.State == client.OpCancelled {
		return nil, fmt.Errorf("operation %s was cancelled", final.OperationID)
	}
	return fetch()
}

// resolveTimeout converts an agent-supplied timeout_seconds (0 = use default)
// into a time.Duration, capped at maxTimeout.
func resolveTimeout(seconds int, defaultDur, maxDur time.Duration) time.Duration {
	if seconds <= 0 {
		return defaultDur
	}
	d := time.Duration(seconds) * time.Second
	if d > maxDur {
		return maxDur
	}
	return d
}
```

type sandboxCreateInput struct {
	ProjectID      string `json:"project_id" jsonschema:"the project to create the sandbox in"`
	ImageID        string `json:"image_id" jsonschema:"the base image ID to launch from"`
	Name           string `json:"name,omitempty" jsonschema:"optional human-readable name"`
	CPULimit       *int   `json:"cpu,omitempty" jsonschema:"optional vCPU limit"`
	MemoryLimitMB  *int   `json:"memory_mb,omitempty" jsonschema:"optional memory limit in MB"`
	Wait           *bool  `json:"wait,omitempty" jsonschema:"wait for the sandbox to reach running state (default true)"`
	TimeoutSeconds int    `json:"timeout_seconds,omitempty" jsonschema:"max seconds to wait (default 300)"`
}

type sandboxIDWaitInput struct {
	SandboxID      string `json:"sandbox_id" jsonschema:"the sandbox's ID"`
	Wait           *bool  `json:"wait,omitempty" jsonschema:"wait for completion (default true)"`
	TimeoutSeconds int    `json:"timeout_seconds,omitempty"`
	Force          bool   `json:"force,omitempty" jsonschema:"force-stop (only meaningful for sandbox_stop)"`
}

func registerSandboxMutatingToolsImpl(s *mcpsdk.Server, opts Options) {
	mcpsdk.AddTool(s, &mcpsdk.Tool{
		Name:        "sandbox_create",
		Description: "Create a new sandbox from a base image. By default waits until the sandbox reaches the running state and returns the final sandbox object.",
	}, func(ctx context.Context, _ *mcpsdk.CallToolRequest, in sandboxCreateInput) (*mcpsdk.CallToolResult, any, error) {
		req := client.CreateSandboxRequest{
			ProjectID: in.ProjectID,
			Name:      in.Name,
			ImageID:   in.ImageID,
		}
		if in.CPULimit != nil {
			req.CPULimit = in.CPULimit
		}
		if in.MemoryLimitMB != nil {
			req.MemoryLimitMB = in.MemoryLimitMB
		}
		op, err := opts.Client.CreateSandbox(ctx, req)
		if err != nil {
			return nil, nil, err
		}
		if in.Wait != nil && !*in.Wait {
			return nil, op, nil
		}
		timeout := resolveTimeout(in.TimeoutSeconds, 300*time.Second, opts.maxTimeout())
		return nil, mustResource(waitForOpAndFetch(ctx, opts.Client, op, timeout, func() (any, error) {
			return opts.Client.GetSandbox(ctx, op.ResourceID)
		}))
	})

	mcpsdk.AddTool(s, &mcpsdk.Tool{
		Name:        "sandbox_start",
		Description: "Start a stopped sandbox. By default waits until it reaches running state.",
	}, func(ctx context.Context, _ *mcpsdk.CallToolRequest, in sandboxIDWaitInput) (*mcpsdk.CallToolResult, any, error) {
		op, err := opts.Client.StartSandbox(ctx, in.SandboxID)
		if err != nil {
			return nil, nil, err
		}
		if in.Wait != nil && !*in.Wait {
			return nil, op, nil
		}
		timeout := resolveTimeout(in.TimeoutSeconds, 120*time.Second, opts.maxTimeout())
		return nil, mustResource(waitForOpAndFetch(ctx, opts.Client, op, timeout, func() (any, error) {
			return opts.Client.GetSandbox(ctx, in.SandboxID)
		}))
	})

	mcpsdk.AddTool(s, &mcpsdk.Tool{
		Name:        "sandbox_stop",
		Description: "Stop a running sandbox. Set force=true for an immediate halt.",
	}, func(ctx context.Context, _ *mcpsdk.CallToolRequest, in sandboxIDWaitInput) (*mcpsdk.CallToolResult, any, error) {
		op, err := opts.Client.StopSandbox(ctx, in.SandboxID, client.StopSandboxRequest{Force: in.Force})
		if err != nil {
			return nil, nil, err
		}
		if in.Wait != nil && !*in.Wait {
			return nil, op, nil
		}
		timeout := resolveTimeout(in.TimeoutSeconds, 60*time.Second, opts.maxTimeout())
		return nil, mustResource(waitForOpAndFetch(ctx, opts.Client, op, timeout, func() (any, error) {
			return opts.Client.GetSandbox(ctx, in.SandboxID)
		}))
	})

	mcpsdk.AddTool(s, &mcpsdk.Tool{
		Name:        "sandbox_destroy",
		Description: "Destroy a sandbox permanently. This cannot be undone.",
	}, func(ctx context.Context, _ *mcpsdk.CallToolRequest, in sandboxIDWaitInput) (*mcpsdk.CallToolResult, any, error) {
		op, err := opts.Client.DestroySandbox(ctx, in.SandboxID)
		if err != nil {
			return nil, nil, err
		}
		if in.Wait != nil && !*in.Wait {
			return nil, op, nil
		}
		timeout := resolveTimeout(in.TimeoutSeconds, 60*time.Second, opts.maxTimeout())
		_, err = waitForOpAndFetch(ctx, opts.Client, op, timeout, func() (any, error) {
			return map[string]bool{"ok": true}, nil
		})
		if err != nil {
			return nil, nil, err
		}
		return nil, map[string]bool{"ok": true}, nil
	})
}

// mustResource adapts (any, error) into (any, error) without losing the value
// when there's no error. Used to keep the generic AddTool signature happy.
func mustResource(res any, err error) (any, error) {
	return res, err
}

package mcp

import (
	"net/http"
	"strings"
	"time"

	mcpsdk "github.com/modelcontextprotocol/go-sdk/mcp"

	"github.com/navaris/navaris/pkg/client"
)

// HTTPOptions configures the embedded HTTP MCP handler.
type HTTPOptions struct {
	// LocalAPIURL is the URL the inner pkg/client.Client uses to call back
	// into navarisd's REST API. Typically "http://localhost:<port>".
	LocalAPIURL string

	// AuthToken is the navarisd bearer token. Inbound requests must present
	// this token in their Authorization header. When empty, no auth is
	// enforced (parity with the test-mode behavior of authMiddleware).
	AuthToken string

	// ReadOnly hides every mutating tool when true.
	ReadOnly bool

	// MaxTimeout caps any per-tool timeout_seconds argument.
	MaxTimeout time.Duration
}

// NewHTTPHandler returns an http.Handler that serves the navaris MCP Streamable
// HTTP transport. Authentication is bearer-token only — session cookies are
// rejected with 401. The bearer token is passed through to the inner client
// so all REST calls inherit the caller's identity.
func NewHTTPHandler(opts HTTPOptions) http.Handler {
	return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
		// Pass-through auth: extract the bearer token, then enforce.
		bearer := extractBearer(r)
		if opts.AuthToken != "" {
			if bearer == "" || bearer != opts.AuthToken {
				w.Header().Set("WWW-Authenticate", `Bearer realm="navaris-mcp"`)
				http.Error(w, "unauthorized", http.StatusUnauthorized)
				return
			}
		}

		handler := mcpsdk.NewStreamableHTTPHandler(func(req *http.Request) *mcpsdk.Server {
			c := client.NewClient(
				client.WithURL(opts.LocalAPIURL),
				client.WithToken(bearer), // pass-through identity
			)
			return NewServer(Options{
				Client:     c,
				ReadOnly:   opts.ReadOnly,
				MaxTimeout: opts.MaxTimeout,
			})
		}, nil)

		handler.ServeHTTP(w, r)
	})
}

func extractBearer(r *http.Request) string {
	h := r.Header.Get("Authorization")
	if !strings.HasPrefix(h, "Bearer ") {
		return ""
	}
	return strings.TrimPrefix(h, "Bearer ")
}

# Navaris MCP Server

The navaris MCP server exposes sandbox/session/snapshot/image management as Model Context Protocol tools so AI agents (Claude Code, Cursor, Claude Desktop, etc.) can drive sandboxes directly.

Two transports are available:

- **stdio** — `navaris-mcp` binary, spawned by the MCP client as a subprocess
- **Streamable HTTP** — `/v1/mcp` endpoint embedded in navarisd, gated by the `--mcp-enabled` flag

Both transports expose the same tool set and read the same `--mcp-read-only` toggle to hide every mutating tool.

## Tool catalog

| Tool | Read-only | Inputs |
|---|:-:|---|
| `project_list` | ✓ | — |
| `project_get` | ✓ | `project_id` |
| `sandbox_list` | ✓ | `project_id`, `state?` |
| `sandbox_get` | ✓ | `sandbox_id` |
| `sandbox_create` |  | `project_id`, `image_id`, `name?`, `cpu?`, `memory_mb?`, `wait?`=true, `timeout_seconds?`=300 |
| `sandbox_start` |  | `sandbox_id`, `wait?`=true, `timeout_seconds?`=120 |
| `sandbox_stop` |  | `sandbox_id`, `force?`, `wait?`=true, `timeout_seconds?`=60 |
| `sandbox_destroy` |  | `sandbox_id`, `wait?`=true, `timeout_seconds?`=60 |
| `sandbox_exec` |  | `sandbox_id`, `command`, `env?`, `work_dir?`, `timeout_seconds?`=120 |
| `session_list` | ✓ | `sandbox_id` |
| `session_get` | ✓ | `session_id` |
| `session_create` |  | `sandbox_id`, `shell?`, `backing?` |
| `session_destroy` |  | `session_id` |
| `snapshot_list` | ✓ | `sandbox_id` |
| `snapshot_get` | ✓ | `snapshot_id` |
| `snapshot_create` |  | `sandbox_id`, `label?`, `consistency_mode?`, `wait?`=true, `timeout_seconds?`=120 |
| `snapshot_restore` |  | `snapshot_id`, `wait?`=true, `timeout_seconds?`=300 |
| `snapshot_delete` |  | `snapshot_id`, `wait?`=true, `timeout_seconds?`=60 |
| `image_list` | ✓ | `name?`, `architecture?` |
| `image_get` | ✓ | `image_id` |
| `operation_get` | ✓ | `operation_id` |
| `operation_cancel` |  | `operation_id` |

Mutating tools default to `wait=true` so the call returns the final resource. Pass `wait=false` to get an `operation_id` you can poll later via `operation_get`. If a `timeout_seconds` elapses while the underlying operation is still running, the tool returns a `{operation_id, status: "running", note: ...}` payload (not an error) so the agent can decide to wait more or move on.

## Stdio (local agents)

The `navaris-mcp` binary reads its config from environment variables. Most MCP clients let you set these in the server config.

| Env | Required | Default | Purpose |
|---|:-:|---|---|
| `NAVARIS_API_URL` | ✓ | — | URL of navarisd (e.g. `http://localhost:8080`) |
| `NAVARIS_TOKEN` | conditional | — | Bearer token; required when navarisd has `--auth-token` set |
| `NAVARIS_MCP_READ_ONLY` |  | `false` | `true` to hide every mutating tool |
| `NAVARIS_MCP_MAX_TIMEOUT` |  | `600s` | Cap on per-tool `timeout_seconds` |
| `NAVARIS_MCP_LOG_FORMAT` |  | `text` | `json` for structured logs to stderr |

## Embedded HTTP (remote/platform agents)

Enable the embedded MCP endpoint in navarisd:

```bash
navarisd \
  --listen :8080 \
  --auth-token your-token \
  --mcp-enabled \
  --mcp-read-only=false \
  --mcp-max-timeout 10m
```

| Flag | Env | Default | Purpose |
|---|---|---|---|
| `--mcp-enabled` | `NAVARIS_MCP_ENABLED` | `false` | Mount `/v1/mcp` |
| `--mcp-read-only` | `NAVARIS_MCP_READ_ONLY` | `false` | Hide all mutating tools |
| `--mcp-path` | `NAVARIS_MCP_PATH` | `/v1/mcp` | Endpoint path |
| `--mcp-max-timeout` | `NAVARIS_MCP_MAX_TIMEOUT` | `10m` | Cap on per-tool `timeout_seconds` |

Authentication: bearer token only. The endpoint sits behind navarisd's existing auth middleware AND additionally rejects requests that lack a bearer token (cookie auth is not accepted on `/v1/mcp` — it is meant for programmatic clients).

The bearer token presented by the MCP client is **passed through** to the inner client used by tool handlers, so audit logs always reflect the caller's identity.

## Read-only mode

When read-only mode is on (`--mcp-read-only` or `NAVARIS_MCP_READ_ONLY=1`):

- Mutating tools (`sandbox_create`, `sandbox_destroy`, `sandbox_exec`, `session_create`, etc.) are NOT registered.
- Agents see only read tools in `tools/list`.
- Calling a missing tool returns `method not found` from the MCP server.

This is enforced at registration time, so there is no per-call gate to forget.

## Limitations (v1)

- No streaming exec, no progress notifications.
- `sandbox_exec` does not accept stdin (deferred — requires changes across all providers).
- No `attach` or session-driving tool — agents use `sandbox_exec` for state-bearing work via tmux sessions they create themselves.
- No image upload, port forwarding, or events subscribe tools.
- The embedded endpoint requires bearer auth (cookie auth, used by the web UI, is rejected here).
- `image_list` does not filter by project (the underlying API doesn't either).
- `snapshot_list` is sandbox-scoped (no project-wide listing).
